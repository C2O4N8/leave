## 用户管理

### 查看用户列表

1. 登录管理员账户
2. 进入「用户管理」页面
3. 查看所有注册用户信息：
   - 用户名
   - 姓名
   - 角色（学生/管理员）
   - 状态（已验证/待验证/已锁定）
   - 注册时间

### 用户状态管理

#### 1. 锁定用户
适用场景：违规用户、毕业学生、暂停使用

操作步骤：
1. 在用户列表找到目标用户
2. 点击「锁定账户」按钮
3. 确认操作
4. 用户状态变为「已锁定」(rejected)

**效果**: 用户被锁定后：
- 无法登录系统
- 如已登录，会自动登出
- 所有功能被禁用

#### 2. 解锁用户
操作步骤：
1. 在用户列表找到被锁定的用户
2. 点击「解锁账户」按钮
3. 确认操作
4. 用户状态变为「已验证」(verified)

**效果**: 用户可以正常登录和使用系统

#### 3. 删除用户
适用场景：测试账户、错误注册

操作步骤：
1. 在用户列表找到目标用户
2. 点击「删除」按钮
3. 确认删除操作
4. 用户及其所有请假记录被删除

⚠️ **警告**: 删除操作不可恢复！

### 用户验证管理

#### 查看待验证用户
筛选条件：status = 'pending'

这些用户：
- 已完成注册

## 秘钥管理

### 秘钥验证机制

系统采用SHA256加密的秘钥验证机制：
1. 管理员为用户生成专属秘钥
2. 秘钥由随机盐值+用户名生成
3. 用户输入秘钥完成验证
4. 每个秘钥只能使用一次

### 生成验证秘钥

#### 步骤：

1. 进入「秘钥管理」页面
2. 点击「生成新秘钥」
3. 输入用户名（必须是已注册的用户）
4. 点击「生成」
5. 系统生成64位秘钥
6. 将秘钥发送给用户

#### 秘钥信息：

生成后显示：
- 用户名
- 生成的秘钥（64位十六进制）
- 生成时间
- 生成者
- 使用状态

#### 秘钥分发方式：

- 面对面告知
- 通过学校邮箱发送
- 通过即时通讯工具发送

⚠️ **安全提示**: 不要在公开场合或不安全的渠道分享秘钥

### 查看秘钥记录

秘钥管理页面显示：
- 所有生成的秘钥
- 使用状态（未使用/已使用）
- 使用时间
- 使用者

可以筛选：
- 未使用的秘钥
- 已使用的秘钥
- 特定用户的秘钥

### 秘钥安全

- 秘钥采用SHA256加密存储
- 每个秘钥只能使用一次
- 使用后立即标记为已使用
- 无法重复使用或撤销

## 数据库维护

### 数据库结构

#### users表（用户表）
```sql
- id: 用户ID
- username: 用户名
- password: 密码（SHA256）
- name, gender, student_id等: 个人信息
- role: 角色（admin/student）
- status: 状态（verified/pending/rejected）
- profile_completed: 信息完善标记
```

#### leaves表（请假表）
```sql
- id: 请假ID
- user_id: 用户ID
- leave_type, destination等: 请假信息
- status: 状态（pending/approved/rejected/invalidated）
- created_at, approved_at: 时间戳
```

#### registration_keys表（秘钥表）
```sql
- id: 秘钥ID
- username: 关联用户名
- generated_key: 秘钥
- is_used: 使用状态
- used_at: 使用时间
```

## 故障排查

### 常见问题

#### 1. 无法连接数据库

**症状**: 启动报错 "数据库连接错误"

**排查步骤**:
```bash
# 1. 检查MySQL服务
net start mysql

# 2. 检查端口
netstat -ano | findstr 3310

# 3. 测试连接
mysql -h 127.0.0.1 -P 3310 -u root -p

# 4. 检查.env配置
cat .env
```

**解决方案**:
- 启动MySQL服务
- 确认端口号正确
- 检查用户名密码
- 确认数据库已创建

#### 2. 用户无法登录

**可能原因**:
- 账户未验证 (status='pending')
- 账户被锁定 (status='rejected')
- 密码错误
- Session过期

**排查SQL**:
```sql
SELECT id, username, status FROM users WHERE username='用户名';
```

**解决方案**:
```sql
-- 验证账户
UPDATE users SET status='verified', verified_at=NOW() WHERE username='用户名';

-- 解锁账户
UPDATE users SET status='verified' WHERE username='用户名';

-- 重置密码
UPDATE users SET password='哈希值' WHERE username='用户名';
```

#### 3. 无法提交请假申请

**可能原因**:
- 个人信息未完善
- 已有待审批申请
- 数据验证失败

**排查SQL**:
```sql
-- 检查信息完善状态
SELECT id, username, profile_completed FROM users WHERE id=用户ID;

-- 检查待审批申请
SELECT * FROM leaves WHERE user_id=用户ID AND status='pending';
```

**解决方案**:
```sql
-- 标记信息为已完善
UPDATE users SET profile_completed=1 WHERE id=用户ID;
```

#### 4. 系统启动失败

**检查项**:
1. Python版本（≥3.8）
```bash
python --version
```

2. 依赖包安装
```bash
pip list | findstr Flask
pip list | findstr mysql-connector
```

3. 端口占用
```bash
netstat -ano | findstr 5000
```

4. 日志错误
查看控制台输出的错误信息

#### 5. 性能问题

**症状**: 系统响应慢

**排查**:
```sql
-- 检查数据量
SELECT COUNT(*) FROM leaves;
SELECT COUNT(*) FROM users;

-- 检查索引
SHOW INDEXES FROM leaves;
SHOW INDEXES FROM users;
```

**优化**:
- 添加索引
- 清理历史数据
- 优化查询SQL
- 增加服务器资源

#### 1. 数据库备份

**每日备份脚本** (`daily_backup.bat`):
```batch
@echo off
set BACKUP_DIR=D:\backup\leave_system
set DATE=%date:~0,4%%date:~5,2%%date:~8,2%

REM 创建备份目录
if not exist %BACKUP_DIR% mkdir %BACKUP_DIR%

REM 备份数据库
mysqldump -h 127.0.0.1 -P 3310 -u root -p你的密码 leave_system > %BACKUP_DIR%\db_%DATE%.sql

REM 删除7天前的备份
forfiles /p %BACKUP_DIR% /m db_*.sql /d -7 /c "cmd /c del @file"

echo Backup completed: %DATE%
```
