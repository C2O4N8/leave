# 请假管理系统 - 系统运行逻辑详解

## 系统架构概述

请假管理系统采用传统的MVC架构，基于Flask框架开发。

### 技术栈
- **后端**: Flask (Python 3.8+)
- **数据库**: MySQL 8.0+
- **前端**: HTML5 + CSS3 + JavaScript
- **UI框架**: Ant Design Mobile

---

## 完整路由列表

### 公开路由（无需登录）

#### 1. `/` - 首页/自动重定向
- **方法**: GET
- **功能**: 根据登录状态重定向到相应页面
- **逻辑**:
  - 未登录 → `/login`
  - 学生已登录 → `/home`
  - 管理员已登录 → `/admin_dashboard`

#### 2. `/login` - 登录页面
- **方法**: GET, POST
- **功能**: 用户登录
- **GET**: 显示登录表单
- **POST**: 验证用户凭据
  - 检查用户名密码（SHA256）
  - 验证账户状态（must be 'verified'）
  - 创建session
  - 重定向到对应首页

#### 3. `/register` - 注册页面
- **方法**: GET, POST
- **功能**: 新用户注册
- **GET**: 显示注册表单
- **POST**: 创建新账户
  - 验证用户名唯一性
  - 密码SHA256加密
  - status设为'pending'
  - 重定向到验证页面

#### 4. `/verify_registration` - 账户验证页面
- **方法**: GET, POST
- **功能**: 验证新注册账户
- **GET**: 显示验证表单
- **POST**: 验证秘钥
  - 检查秘钥是否存在且未使用
  - 验证成功：status改为'verified'
  - 标记秘钥为已使用
  - 重定向到登录页

### 学生路由（需登录且role='student'）

#### 5. `/home` - 学生首页
- **方法**: GET
- **功能**: 学生主页面
- **显示内容**:
  - 欢迎信息
  - 快捷功能入口
  - 最近请假记录
  - 底部导航栏

#### 6. `/home_apps` - 应用列表
- **方法**: GET
- **功能**: 显示所有可用应用
- **包含功能**:
  - 学生请假（点击进入请假列表）
  - 其他应用模块
- **特殊逻辑**:
  - 点击"学生请假"进入 `/h_manage`
  - 不设置sessionStorage标记（已移除from_app逻辑）

#### 7. `/user_dashboard` (别名 `/dashboard`)
- **方法**: GET
- **功能**: 用户仪表板
- **显示内容**:
  - 个人统计数据
  - 快捷功能卡片
- **特殊逻辑**:
  - 点击"我的请假"设置 `hide_app_crumb=true`
  - 进入列表页后不显示"应用>"面包屑

#### 8. `/profile` - 个人信息管理
- **方法**: GET, POST
- **功能**: 查看和编辑个人信息
- **GET**: 显示当前信息
- **POST**: 更新信息
  - 更新姓名、性别、学号等
  - 设置 `profile_completed=1`
  - 首次完善后才能提交请假

#### 9. `/apply` - 请假申请页面
- **方法**: GET
- **功能**: 显示请假申请表单
- **前置条件**:
  - 必须已完善个人信息
  - 不能有待审批的申请
- **表单字段**:
  - 请假类型、时间、去向
  - 紧急联系人
  - 请假理由

#### 10. `/apply_submit` - 提交请假申请
- **方法**: POST
- **功能**: 处理请假申请提交
- **验证逻辑**:
  - 检查profile_completed
  - 检查是否有pending申请
  - 验证时间范围
  - 验证必填字段
- **成功后**: 重定向到 `/h_manage`

#### 11. `/h_manage` - 请假列表
- **方法**: GET
- **功能**: 显示所有请假记录
- **查询参数**: `?status=` (pending/approved/rejected/invalidated)
- **面包屑逻辑**:
  - 默认显示"应用>"
  - 如果sessionStorage有`hide_app_crumb=true`则隐藏
  - 点击"应用>"返回 `/home_apps`
- **功能**:
  - 状态筛选
  - 查看详情
  - 提交新申请入口

#### 12. `/detail/<int:leave_id>` - 请假详情
- **方法**: GET, POST
- **功能**: 查看单个请假申请详情
- **显示内容**:
  - 完整请假信息
  - 审批进度
  - 审批人信息
  - 审批备注
- **面包屑逻辑**:
  - 继承list页面的hide_app_crumb状态
  - 点击"请假管理"返回列表

#### 13. `/import_history` - 导入历史记录
- **方法**: GET, POST
- **功能**: 从HTML文件导入历史请假记录
- **GET**: 显示上传表单
- **POST**: 处理文件上传
  - 解析HTML文件
  - 提取请假记录
  - 批量导入数据库
  - 显示导入结果

#### 14. `/user_settings` - 用户设置
- **方法**: GET
- **功能**: 用户个人设置
- **包含**:
  - 安全模式配置
  - 其他个性化设置

## 核心业务逻辑

### 1. 用户认证流程

```
注册 → 待验证(pending) → 秘钥验证 → 已验证(verified) → 登录成功
```

**详细步骤**:
1. 用户填写注册表单
2. 系统创建账户，status='pending'
3. 管理员生成验证秘钥
4. 用户输入秘钥验证
5. 验证成功，status='verified'
6. 用户可以登录系统

### 2. 请假申请流程

```
完善信息 → 提交申请(pending) → 管理员审批 → approved/rejected
```

**详细步骤**:
1. 学生首次登录完善个人信息
2. profile_completed设为1
3. 填写请假申请表单
4. 提交后status='pending'
5. 管理员在审批列表查看
6. 批准：status='approved'
7. 拒绝：status='rejected'

### 3. 用户状态管理

**三种状态**:
- **pending**: 待验证，无法登录
- **verified**: 已验证，可正常使用
- **rejected**: 已锁定，自动登出

**状态转换**:
- pending → verified: 秘钥验证
- verified → rejected: 管理员锁定
- rejected → verified: 管理员解锁

### 4. 面包屑导航逻辑

**规则**:
- 默认显示: 首页 > 应用 > 请假管理 > 当前页
- 从user_dashboard进入: 首页 > 请假管理 > 当前页（隐藏"应用>"）
- 从home_apps进入: 首页 > 应用 > 请假管理 > 当前页

**实现机制**:
- user_dashboard设置 `sessionStorage.hide_app_crumb='true'`
- leave_list.html检查标记，隐藏"应用>"
- leave_detail.html继承hide_app_crumb状态
- 点击"应用>"返回home_apps时清除标记

### 5. 安全模式功能

**目的**: 快速切换到目标网站（如学校系统）

**实现**:
- 用户在设置中开启安全模式
- 设置目标网站URL
- 点击"首页"时检查安全模式
- 如果启用，直接跳转到目标网站
- 利用浏览器管理Cookie，无需后端代理

## 数据库设计

### users表关键字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| username | VARCHAR(100) | 用户名（唯一） |

| role | VARCHAR(20) | admin/student |
| status | VARCHAR(20) | verified/pending/rejected |
| profile_completed | TINYINT | 0/1 信息完善标记 |
| verification_key | VARCHAR(255) | 验证秘钥 |

### leaves表关键字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| user_id | INT | 外键关联users.id |
| leave_type | VARCHAR(50) | 请假类型 |
| start_time | DATETIME | 开始时间 |
| end_time | DATETIME | 结束时间 |
| status | VARCHAR(20) | pending/approved/rejected/invalidated |
| approver_name | VARCHAR(100) | 审批人姓名 |
| approved_at | DATETIME | 审批时间 |

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| username | VARCHAR(100) | 关联用户名 |
| salt | VARCHAR(255) | 随机盐值 |
| generated_key | VARCHAR(255) | SHA256秘钥 |
| is_used | TINYINT | 0/1 使用状态 |
| used_at | DATETIME | 使用时间 |

## 安全机制

### 1. 密码加密
- 使用SHA256哈希算法
- 不存储明文密码
- 每次登录重新计算哈希对比

### 2. Session管理
- Flask内置Session机制
- Cookie httponly标记
- Secret key加密
- 自动过期清理

### 3. SQL注入防护
- 使用参数化查询
- 不直接拼接SQL字符串
- mysql-connector-python自动转义

### 4. XSS防护
- Jinja2模板自动转义
- 用户输入过滤
- HTML特殊字符处理

### 5. CSRF防护
- 所有POST请求检查来源
- Session token验证
- 重要操作需要确认

### 6. 权限控制
- 路由级别权限检查
- 学生/管理员分离
- 锁定用户自动登出
